# Output Template

Use this section order unless the user explicitly asks for another format.

## 1. Executive Summary

Include:

- target rebuild objective
- biggest preserved function
- biggest avoided risk
- biggest remaining uncertainty

## 2. Rebuild Brief Snapshot

List:

- upstream package used
- target market
- dosage form
- must-keep cues
- must-avoid elements
- desired benefit directions

## 3. Functional Goal Map

| functional_goal | upstream_supporting_item | protection_status | rebuild_note |
| --- | --- | --- | --- |

## 4. Unsafe-to-Safe Substitution Map

| risky_item | role_in_target | upstream_posture | safer_direction | notes |
| --- | --- | --- | --- | --- |

## 5. Candidate Rebuild Directions

For each direction, include:

- short direction name
- core function preserved
- category cues kept
- risky elements intentionally avoided
- claim posture
- commercialization notes

## 6. Claim Guardrails

List:

- safer naming choices
- claims to keep conservative
- claims or wording to avoid
- documentation likely needed before launch

## 7. Residual Risks & Required Review

List the remaining high-value checks before commercialization.
