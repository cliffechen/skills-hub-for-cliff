# Workflow

## 1. Validate the Upstream Package

Prefer these upstream sections:

- `Observed Ingredient Table`
- `Ingredient Classification`
- `Risk Matrix`
- `Missing Evidence Points`
- `Rebuild Posture Summary`

If something is missing:

- name the missing section
- explain why it matters
- carry uncertainty forward

## 2. Split Function From Protected Implementation

For each key item, separate:

- consumer need
- functional goal
- category cue
- ingredient family
- branded or proprietary implementation
- source or process story
- claim language

## 3. Build the Fixed Outputs

### Functional Goal Map

Map:

- what the user wants to preserve
- which upstream item supports it
- whether that support is generic or protected
- how to preserve it conservatively

### Unsafe-to-Safe Substitution Map

For each risky item, ask:

1. What role is it playing?
2. Is the role generic or owner-specific?
3. Can the same role be rebuilt with a more generic ingredient family or simpler support logic?
4. Does the claim need to be narrowed even if the ingredient changes?

### Candidate Rebuild Directions

Default to 2 to 3 directions:

- conservative generic direction
- differentiated direction
- compliance-first direction

## 4. Output Rules

- Answer in Chinese.
- Keep exact English branded names or Latin source names where necessary.
- Focus on safer direction, not legal certainty.

## 5. Guardrails

- Do not recommend another party's branded ingredient name.
- Do not recreate a competitor's exclusive source story.
- Do not use disease, treatment, reversal, or superiority claims to compensate for formula changes.
- Do not produce a clone-ready manufacturing recipe.
