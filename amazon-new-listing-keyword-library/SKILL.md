# 亚马逊新品关键词词库搭建系统

为亚马逊(默认美国站)新品搭建关键词词库:多渠道挖词 → 合并去重 → 相对化分级(P0/P1/P2) → 落地映射 → 输出词库Excel + 选词策略报告。打法基调默认**精品/天机打法**(集中抢5-6个核心词)。

完整方法论背景见 `references/` 目录(开工前建议通读):

- `references/项目思路总览.png` — 整体流程图(四渠道挖词逻辑)
- `references/关键词收集术.md` — 合并/去重/分级/落地四步法与P0P1P2标准
- `references/整理一份亚马逊广告打法全策略手册_广告心得.md` — 天机打法、坑位监控、CDQ/PQS/SCS等
- `references/【listing优化】2026亚马逊listing优化适配Alexa指南.docx` — Alexa/COSMO语义优化(用脚本解zip读document.xml,python-docx可能因畸形zip报错)
- `references/POPE-Listing打造SOP-截图.png` — Listing埋词结构SOP

输入文件的格式样例见 `examples/input-templates/`,成品参照见 `examples/sample-output/`。

## 可调参数(按类目调整,本节数值为默认)

- ABA命脉词阈值:种子ASIN合计点击份额 ≥ 30%
- ABA无人区词阈值:份额 < 10% 且种子在Top3内(注意:种子不在Top3 ≠ 份额低,数据缺失不可判为无人区)
- 天机核心词:相关流量池内累计占比 70-80% 的头部非品牌词,5-6个
- P0数量 10-20;P1数量 50-100;P2不限
- Search Term 上限 249 字节

## 前置澄清(开工前用提问工具确认)

1. 打法基调:精品/天机(默认) vs 海量收录 vs 两者结合
2. 输出格式:Excel词库 + Markdown报告(默认)
3. 产品卖点信息不足时:从竞品标题反推并标注假设(默认) vs 用户补充
4. 站点(默认US)

## 第0步:输入识别与校验

按**内容特征**(非文件名)识别4类输入:

1. **ABA品牌分析CSV**:首行含 `点击量最高的商品=["ASIN1 ASIN2 ..."]` 元数据(即种子ASIN),列含"搜索频率排名/搜索词/点击份额"。注意 `utf-8-sig` 编码、首行元数据需单独读取后再用DictReader。
   **⚠️ 列名不一致陷阱(实测踩坑,DictReader取列必须用下面的实际列名,不要按规律猜)**:同一组Top3列的命名在1/2/3名之间不统一——
   | 字段 | 第1名 | 第2名 | 第3名 |
   |---|---|---|---|
   | ASIN | `点击量第1的商品：ASIN` | `点击量第2的商品：ASIN` | `点击量第**三**的商品：ASIN`(汉字三) |
   | 点击份额 | `点击量**最高**的商品：点击份额` | `点击量**第二**的商品：点击份额` | `点击量**第三**的商品：点击份额` |
   | 商品标题 | `点击量第1的商品: 商品标题`(**半角冒号+空格**) | `点击量第2的商品：商品标题` | `点击量第3的商品：商品标题` |
   | 转化贡献 | `点击量第1的商品：转化贡献占比` | `**热门点击商品第2名**：转化贡献占比`(整句式不同) | `点击量第3的商品：转化贡献占比` |
   | 分类 | `点击量最高的分类` | `点击量第二的分类` | `点击量第 3 的分类`(数字带空格) |

   稳妥做法:解析前先打印实际表头逐列核对;或按"列位置索引"+表头模糊匹配(归一化全/半角冒号与空格后再matching)取数,不要硬编码猜测的列名。不同站点/语言导出的列名还可能再变,核对表头这步不可省。
2. **Sif关键词调研xlsx**:双sheet结构——"高、中、低相关(N个)" 和 "几乎不相关(N个)";列含 关键词/中文翻译/相关性/相关性得分/周搜索量/ABA排名/点击转化率/建议竞价(低中高)/Top3点击集中度/Top3转化集中度/Top4-48占位率。
3. **下拉搜索框txt**:纯词列表,一行一词。
4. **产品信息图**(Supplement Facts等):读图提取成分、剂量、规格、形态(softgel/capsule/gummy/powder)、辅料(注意:辅料决定vegan等属性词的真伪,如牛明胶=非素食)。

缺失任何一类 → 提示用户补交或确认跳过。卖点信息不足时,从ABA/Sif中竞品标题反推产品画像,**所有反推假设在报告中显式标注**。

## 第1步:分渠道挖词

**ABA**:计算每词的种子ASIN合计点击份额。≥30% = 命脉词(必埋必打);<10%且种子在Top3 = 无人区词(只埋不打);其余进总库。

**Sif**:只用"高、中、低相关"sheet。剔除品牌词后,按周搜索量计算池内累计流量占比,锁定占比70-80%的5-6个核心词。**警惕Sif默认排序陷阱**:相关性得分最高 ≠ 该选,要用 搜索量 × 相关性 × 可抢性 综合判断。Top3点击集中度>70%的词标记为垄断词(通常是品牌词),降级处理。
"几乎不相关"sheet做防误杀抽查:筛 sv≥500 且语义沾边的词人工复核(注意成分不符的词如复配词,单方产品不收)。

**下拉框**:去重后按意图分5类——品牌词/形态词/剂量词/复配词/通用长尾。品牌词进否定清单;**形态错配词**(与我方产品形态不符,如卖softgel遇到gummies/powder)只埋Search Term不投广告(转化错配伤CDQ);与我方属性匹配的高意图词(认证tested/gmp/nsf、剂量、人群men/women、形态匹配)提升P1。

## 第2步:数据相对性处理(两层相对化,核心方法论)

绝对阈值跨类目必然失效(蓝海子类周搜10000是大词,热门成分类10000是小词),一律改用:

**第一层 市场内百分位**(用户文件,零成本):分级基于词池内的相对位置(累计流量占比、池内排序),不用绝对搜索量阈值。

**第二层 市场级定标**(Sorftime MCP,按需调用;**未配置Sorftime时跳过此层,报告中标注"市场定标数据缺失",其余流程不受影响**):回答"这个市场值不值得打、新品能不能进"。

Sorftime触发条件(5条,避免无脑全量调用消耗配额):
1. 用户文件缺失/字段不全时兜底补数
2. P0候选词逐词查竞争透视(`keyword_detail`:首页非BS占比、评价数<100/300/500分布、CPC、旺季、Top品牌垄断度)——**"首页评价<500产品占比高"= 新品窗口大,是可抢性的硬证据**
3. 两数据源数字打架时仲裁(Sif与Sorftime搜索量不同源,以区间呈现,不混用绝对值;Sif做池内排序、Sorftime做定标)
4. 无搜索量数据的关键词批量补数
5. 市场级定标:`keyword_trend`看24个月趋势(判断是炒作期还是稳态期)、季节性旺季定推新时间锚点、与大盘成分词对比定位蓝海/红海

## 第3步:合并去重分级

统一小写/去空格/单复数,多来源词加权(来源数≥2优先)。分级:

- **P0**(10-20个):核心流量词 + 与我方规格强绑定的规格词。条件 = 高相关 × 池内头部 × 非垄断(可抢)。竞品垄断的大词(如品牌词)即使流量再大也不进P0。
- **P1**(50-100个):品类功效词(往往CPC便宜,低成本流量入口)、属性匹配长尾(认证/剂量/人群/形态/工艺)。
- **P2**:其余相关长尾 + 形态错配词(单独标注)。
- **品牌词否定清单**:全部竞品品牌词(含拼写变体如mitapure/mitipure)。垄断词标注"中后期(评价300+)可ASIN定投+低价词组进攻"。

## 第4步:落地映射

每词标注动作:标题(仅3-4个P0,自然融入)/五点(每条1-2个P0/P1)/A+(P1/P2场景词)/Search Term(249字节内,**剔除标题五点已占用词根**、不含品牌词、空格分隔无逗号,拼好成品直接可粘贴并校验字节数)/广告组归属。

Alexa/COSMO适配:额外产出语义场景短语组(意图短语,如"mitochondrial function support for adults over 50"),用于五点/A+,目标是占据知识图谱节点而非堆词根。

## 第5步:输出

1. **词库xlsx**(保存到用户工作文件夹),6个sheet:关键词总库(全量+分级+来源+数据+动作)/P0核心词-逐词分析(含Sorftime竞争透视列)/P1拓展词(分类型:品类功效/信任认证/剂量/人群/形态/工艺/比价意图)/P2织网词(形态错配单独标注)/品牌词与否定清单/SearchTerm成品(标注字节数)。
2. **选词策略报告md**,必含章节:产品画像(标注反推假设与需人工确认项)、市场基准(相对性定标:规模/趋势/季节/竞争结构/垄断度/CPC水位+数据口径说明)、选词策略总述(三层漏斗)、P0逐词理由(为什么选,以及高流量词为什么不选)、新品打法三阶段节奏(收录期→抢排名期(旺季前锚点)→守位拓量期)、广告策略方向(结构/流量/否定/节奏/预算倾斜原则,**不给具体竞价**)、词库使用与维护节奏(周回灌出单词/月反查竞品/季审核分级)。
3. 输出后做**最终验证**(必须执行,不可跳过):
   - **Search Term 字节校验(强制)**:用 Python 脚本逐字节验证 SearchTerm 成品 sheet 中拼接好的 search_term 字符串:
     ```python
     st = "<拼好的Search Term成品>"
     byte_len = len(st.encode("utf-8"))
     assert byte_len <= 249, f"Search Term 超限: {byte_len} bytes > 249"
     print(f"✅ Search Term 字节数: {byte_len}/249")
     ```
     若超限,必须裁剪末尾词后重新校验,直至 ≤249 bytes 才能写入 xlsx。
   - 重新打开 xlsx 核对各 sheet 行数与分级计数一致。
   - 确认无品牌词混入 P0/P1。

## 关键判断原则(报告灵魂)

- 预算向**可抢性**最高的词倾斜,而非搜索量最大的词
- 品牌词流量"看得见抢不着",计算核心流量占比时必须先剔除
- 垄断词冷启动不打,留作中后期ASIN定投进攻
- 形态错配词只收录不投放
- 旺季是时间锚点:倒推冷启动必须完成的节点
- 词库是活的:交付时附维护节奏
