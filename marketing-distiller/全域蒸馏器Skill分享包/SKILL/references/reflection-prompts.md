# Reflection Prompts — 每步复盘 prompt 模板

每一步执行完毕后，将对应的复盘 prompt 追加到该步主输出文件末尾，同时汇总到 `_REFLECTION.md`。

用户可以直接复制这些 prompt 给任意 LLM（包括下一次对话中的我）进行二次追问。

---

## 通用格式

每段复盘 prompt 固定格式：

```markdown
---

> 💡 **[Step N] 复盘 Prompt**
>
> 复制以下内容给 AI 进行二次追问：
>
> ```
> {具体的追问 prompt}
> ```
```

## Step 0 初始化复盘

```markdown
---

> 💡 **Step 0 — 初始化复盘**
>
> ```
> 我正在研究 [{target}]，已经完成了初始化：
> - 判定 mode = {mode}
> - 深度档位 = {depth}
> - 命中的骨架 = {skeleton_name 或 generic}
> - 输出目录 = {output_path}
>
> 请帮我检查：
> 1. mode 判定是否合理？是否应该调整为其他模式？
> 2. 有没有遗漏的相关行业/概念应该纳入研究范围？
> 3. 深度档位是否合适？有没有需要特别加深的维度？
> 4. 这个研究目标是否需要拆分成多个子目标分别蒸馏？
> ```
```

## Step 1 建骨架复盘

```markdown
---

> 💡 **Step 1 — 建骨架复盘**
>
> ```
> 我正在研究 [{target}]，已完成骨架搭建：
> - mode = {mode}
> - 目录结构已创建：{列出的主要目录}
> - 使用的骨架基础：{skeleton_name}
>
> 请帮我检查：
> 1. 这个行业的目录结构是否有重大遗漏？（比如缺少了某个重要的子领域）
> 2. 对于 [{target}] 这种类型的研究对象，有没有更适合的分类方式？
> 3. 是否需要增加或删减某些子目录来更好地组织信息？
> 4. 你觉得这个行业最重要的 3 个分析维度是什么？我是否都覆盖到了？
> ```
```

## Step 2 填数据库复盘

### 2.1 品牌库复盘

```markdown
---

> 💡 **Step 2.1 — 品牌库复盘**
>
> ```
> 我正在研究 [{target}] 的品牌格局，已整理了 {N} 个品牌：
> - Top 3: {列出前3}
> - 数据完整性：{X}% 的品牌有完整字段
> - 缺失最多的字段：{列出}
> - 数据截止日期：{date}
>
> 请帮我检查：
> 1. 品牌名单有没有明显的遗漏？特别是：
>    - 新兴品牌/初创公司是否被忽略？
>    - 海外/跨区域品牌是否覆盖？
>    - 已倒闭但仍有参考价值的品牌是否该列入？
> 2. 这些品牌的排名/分类是否合理？
> 3. 有哪些重要品牌我完全没提到？
> 4. 对于缺失数据的字段，你知道哪些可靠的信息来源可以补充？
> ```
```

### 2.2 产品库复盘

```markdown
---

> 💡 **Step 2.2 — 产品库复盘**
>
> ```
> 我正在研究 [{target}] 的产品体系，已建立分类树：
> - 一级分类 {N} 个：{列出}
> - 二级分类 {N} 个
> - 叶子节点（具体产品类别）{N} 个
>
> 请帮我检查：
> 1. 产品分类体系是否完整？有没有重要的产品类型被遗漏？
> 2. 分类层级是否合理？有些类别是否应该合并或拆分？
> 3. 最近 1-2 年出现的新产品/新品类是否有位置？
> 4. 这个分类体系和行业内通用的分类法一致吗？如果不一致，是我的更好还是通用的更好？
> ```
```

### 2.3 痛点库复盘

```markdown
---

> 💡 **Step 2.3 — 痛点库复盘**
>
> ```
> 我正在研究 [{target}] 的用户痛点，已整理：
> - 高频抱怨 {N} 条（最高 ⭐{N}）
> - 高频需求 {N} 条
> - FAQ {N} 条
> - 用户目标分 {N} 层
>
> 请帮我检查：
> 1. 这些痛点是否真正反映了用户的心声？有没有"我觉得应该是痛点"但实际上用户不在意的？
> 2. 有哪些真实存在的用户痛点被我漏掉了？特别是：
>    - 新手用户的痛点？
>    - 进阶/专业用户的痛点？
>    - 特殊人群（如企业用户 vs 个人用户）的痛点？
> 3. 痛点的优先级排序是否合理？你认为最值得解决的前 3 个是什么？
> 4. 这些痛点中有哪些是被过度解决的（方案太多）？哪些是几乎没人做的蓝海？
> ```
```

### 2.4 关键词库复盘

```markdown
---

> 💡 **Step 2.4 — 关键词库复盘**
>
> ```
> 我正在研究 [{target}] 的关键词体系，已整理：
> - Commercial 类 {N} 个
> - Informational 类 {N} 个
> - Comparison 类 {N} 个
> - Review 类 {N} 个
> - Problem 类 {N} 个
> - 长尾机会词 {N} 个
>
> 请帮我检查：
> 1. 这些关键词是否全面覆盖了用户的搜索行为？
> 2. 有没有高价值但我没收录的关键词？
> 3. 长尾机会词中，哪些你认为真正的竞争度低且商业价值高？
> 4. 不同平台（Google/百度/小红书/知乎/Amazon）的用户搜索习惯差异是否体现在关键词分类中了？
> 5. 最近 6 个月新出现的搜索热词有哪些我没捕捉到的？
> ```
```

## Step 3 竞品拆解复盘

```markdown
---

> 💡 **Step 3 — 竞品拆解复盘**
>
> ```
> 我正在研究 [{target}]，已完成竞品拆解：
> - 分析竞品数量：{N} 个（{列出名字}）
> - 竞品类型分布：直接竞品{N} / 间接竞品{N} / 新兴挑战者{N} / 跨区域对标{N}
>
> 发现的行业共识（≥60%竞品都在做）：{列出2-3条}
> 发现的差异化机会（≤20%竞品在做）：{列出2-3条}
>
> 请帮我检查：
> 1. 我的竞品选择是否合理？有没有非常重要的竞品被遗漏？
> 2. 竞品分析的深度够不够？哪些维度的分析你觉得最有价值，哪些可以简化？
> 3. 「行业共识」的判断是否准确？有没有哪些看似共识但其实并非如此？
> 4. 「差异化机会」中，哪些你认为真的有机会？哪些可能是别人不做因为确实没用？
> 5. 如果你要进入这个市场，你会选择模仿哪个竞品的哪部分？为什么？
> ```
```

## Step 4 内容生态复盘

```markdown
---

> 💡 **Step 4 — 内容生态复盘**
>
> ```
> 我正在研究 [{target}] 的内容生态，已完成：
> - 采样账号 {N} 个，覆盖平台：{列出}
> - 爆款内容归类：Exposure{N} / Growth{N} / Save{N} / Conversion{N} / Personal{N}
> - 发现的重复爆款话题 Top 5：{列出}
> - 内容空白点：{列出}
> - 内容战略矩阵已生成
>
> 请帮我检查：
> 1. 我采样的账号覆盖面够不够？有没有重要的内容创作者/KOL/媒体被遗漏？
> 2. 内容五分类的使用是否准确？有没有被误判的类型？
> 3. 「重复爆款」的分析结论是否符合你对这个行业的观察？
> 4. 内容空白点真的是空白吗？还是其实有人做但没火起来？
> 5. 如果让你在这个行业做内容，你会选择哪个赛道切入？用什么内容策略？
> 6. 我的内容战略矩阵评估（高确定性成功 vs 低确定性成功）是否靠谱？
> ```
```

## Step 5 地图+机会复盘

```markdown
---

> 💡 **Step 5 — 知识地图 & 机会复盘**
>
> ```
> 我正在研究 [{target}]，已完成知识地图和机会分析：
> - 知识地图：一级{N}领域 / 二级{N}方向 / 三级{N}节点
> - 知识卡片：{N}张（Deep档）/ （Standard档可选）
> - 识别机会：{N}个（🟢蓝海{N} / 🟡成长{N} / 🔴红海{N} / ⚪新兴{N}）
> - 推荐TOP 5：{列出}
>
> 请帮我检查：
> 1. 知识地图的结构是否合理？一个完全不了解这个行业的人看完能否建立基本认知？
> 2. 有没有重要的细分领域/概念/玩家在地图中没有位置？
> 3. 机会推荐 TOP 5 的推理链是否成立？有没有过度乐观的判断？
> 4. 有哪些我认为不是机会但你认为其实是的机会？（反之亦然）
> 5. 这些机会的时间窗口如何？哪些是有时效性的（需尽快行动）？
> 6. 从风险角度看，最大的 3 个风险是什么？
> ```
```

>
> ```
> 我正在为 [{target}] 配置持续情报系统，已完成：
> - 信息源 {N} 个（L1必读{N} / L2选读{N} / L3备查{N}）

> - 内容趋势监控项 {N} 个

> 2. 信息源的分层（L1/L2/L3）是否合理？
> 3. 监控方案的可行性如何？哪些项在实际操作中最容易执行？哪些最容易中断？
> 4. 监控频率是否合适？会不会太多导致噪音，或者太少导致遗漏？
> 5. 除了这些监控手段，还有哪些低成本的方式可以保持对这个行业的敏感度？

```markdown

## 🪞 复盘 prompt

我完成了「{{INPUT}}」行业的 URL 验证清洗，结果在: `Competitors/_VERIFIED_URLS.md`

验证数据:
- 总候选 URL: {{N_TOTAL}}
- ✅ 通过验证: {{N_VERIFIED}}
- ❌ 验证失败: {{N_FAILED}}
- ⚠️ 需人工确认: {{N_REVIEW}}
- 验证方法: multi-source-cross-check (Google + LinkedIn + Wikipedia/Crunchbase)

请审查：

1. **遗漏品牌**：有没有这个行业里**非常重要但我连候选URL都没找到**的品牌？（特别是：B2B渠道为主不打广告的 / 私域/家族企业/非英语市场的）
2. **误判风险**：我标记为 ✅ verified 的 URL 里，有没有可能是**代理页/分销商/冒牌站**？你建议我再抽查哪几个？
3. **失败原因分析**：那 {{N_FAILED}} 个失败URL，是**真的找不到官网**还是**我的搜索策略有盲区**？如果是后者，我应该换什么关键词/什么语法？
4. **地域偏差**：我是不是只找到了美国/英语市场的官网？漏了哪些重要区域市场？（欧盟/日本/东南亚/拉美/中国）

给我 5 个补抓品牌名 + 你建议用的搜索语法（直接给 `site:` 或 `intitle:` 的精确查询）。
```

```markdown
---
title: 复盘 Prompts 汇总
slug: {slug}-OS-reflection
created: {YYYY-MM-DD}
tags: [reflection, {slug}-os]
---

# {target} - 复盘 Prompt 汇总

> 将以下任意一段复制给 AI，即可针对该步骤进行二次追问和深化分析。

## 使用方法
1. 选择你想深入复盘的 Step
2. 复制 ``` 代码块内的内容
3. 发送给任意 LLM（推荐再次使用 Distiller skill 时附带）

---

## Step 0 — 初始化复盘
{粘贴上面的 Step 0 复盘 prompt}
