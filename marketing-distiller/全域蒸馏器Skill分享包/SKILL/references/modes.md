# Modes — 4 种模式的详细差异化定义

本文档定义 industry / brand / website / topic 四种模式的完整差异，包括字段裁剪、取数策略、输出范围。

---

## 一、mode = industry（行业研究）

**适用**: 用户输入的是宽泛的行业/市场/赛道名称。
**示例**: `"美国减肥补充剂市场"`, `"AI Coding 工具"`, `"中国新能源汽车"`, `"DTC 电商"`

### 执行范围
- ✅ Step 1: 完整骨架（全目录）
- ✅ Step 2: 全部 5 个子数据库（品牌/产品/痛点/关键词/内容源）
- ✅ Step 3: Top 5-10 竞品拆解（覆盖直接竞品+间接竞品+新兴挑战者+跨区域对标）
- ✅ Step 4: 全平台内容生态采样（100账号 Standard / Deep 全量）
- ✅ Step 5: 完整知识地图 + 卡片 + 连接关系 + 机会清单
- ⚪ Step 6: 仅 Deep 档执行

### 特殊处理
- 行业边界可能模糊时，在 config 中明确「包含什么」「不包含什么」
- 如果行业太大（如"消费品"），建议用户缩小范围或自行划分子行业
- 关注上下游产业链关系（Step 5.3 连接关系图重点做）

### 取数策略
- 品牌库: 全球视野，中英文源都要搜
- 竞品选择: 至少包含 1-2 个海外对标（即使是中国行业）
- 内容生态: 国内外平台都要采样（国内=小红书/抖音/公众号/知乎；海外=YT/X/TikTok/IG/Reddit）
- 关键词: 中英文双语

---

## 二、mode = brand（品牌拆解）

**适用**: 用户输入的是一个具体的公司或产品品牌名。
**示例**: `"OpenAI"`, `"Lululemon"`, `"瑞幸咖啡"`, `"小米"`, `"Shopify"`

### 执行范围
- ✅ Step 1: 只建 Brands/, Content/, Competitors/ 子集（不需要完整行业骨架）
- ✅ Step 2: 该品牌的产品数据库 + 用户痛点（围绕该品牌产品的痛点）
- ✅ Step 3: **深拆该品牌本身**（作为主分析对象）+ 选取 3-5 个直接竞品对比
- ✅ Step 4: 该品牌的内容矩阵（官方账号+用户生成内容）
- ✅ Step 5: 以该品牌为中心的知识地图 + 相对位置分析
- ⚪ Step 6: 仅 Deep 档执行

### 特殊处理
- **第一步回溯**: 先确定该品牌属于哪个行业，以行业为坐标系再展开分析
- 品牌作为「圆心」，其他品牌作为背景参照系
- 竞品拆解时，该品牌的分析深度 > 其他竞品（其他竞品做轻量版）

### 字段裁剪
- 品牌库: 主品牌卡片超详实（所有字段填齐），竞品卡片精简版
- 产品库: 重点做主品牌的全量产品线，竞品只做重叠/竞争产品
- 痛点库: 围绕主品牌产品的用户评价/投诉/讨论
- 关键词: 品牌词 + 竞品对比词 + 品牌相关问答词

### 取数策略
- 主品牌信息: 官网 + Wikipedia + 新闻 + 财报 + 社媒全平台扫描
- 竞品信息: 快速对比维度即可（价格/功能/定位/渠道）
- 内容: 官方社媒 + KOL评测 + 用户UGC + 新闻报道

---

## 三、mode = website（网站拆解）

**适用**: 用户输入的是一个 URL 或域名。
**示例**: `"https://shop.goli.com"`, `"notion.so"`, `"xiaohongshu.com"`

### 执行范围
- ✅ Step 1: 仅建 Competitors/ 目录（极简骨架）
- ⏭️ Step 2: 跳过数据库建设
- ✅ Step 3: **深拆该 URL**（全维度网站反向工程）+ 自动识别同类型站点作为竞品
- ⚪ Step 4: 可选（如果该网站是内容型站点则跑，否则跳过）
- ✅ Step 5: 仅产出站点级别的机会清单（SEO机会/UX改进/转化优化/内容缺口）
- ⚪ Step 6: 仅 Deep 档执行

### 特殊处理
- **自动识别**: 先通过 WebFetch 抓首页判断这是什么类型的网站（电商/SaaS/内容媒体/社区/工具），再决定后续策略
- 竞品池: 从该网站的行业属性出发，自动搜索同类网站
- 不回溯到完整行业（除非用户要求）

### 字段裁剪
- 导航栏: 全量拆解（这是核心产出）
- Collection/分类: 全量分析
- Tag体系: 全量统计
- SEO: 全量分析（title/meta/h1/internal-links）
- Blog/内容: 如果有则全量分析
- 社媒矩阵: 扫描 footer + about 页中的链接
- 收入推断: 通过 traffic estimate / pricing page / 产品数量推断

### 取数策略
- WebFetch 为主工具（首页/产品页/Blog/About/Pricing/Cases）
- WebSearch 补充: "{domain} 竞品 similar sites alternatives"
- WebSearch: "{domain} traffic revenue 估计" / "{domain} 月活 用户量"

## 四、mode = topic（概念/技术研究）

**适用**: 用户输入的是一个窄概念、技术词汇、或具体话题。
**示例**: `"GLP-1 减肥药"`, `"MCP 协议"`, `"向量数据库"`, `"RAG 技术"`, `"碳中和交易"`

### 执行范围
- ✅ Step 1: 精简骨架（Knowledge-Cards/ + Pain-Points/ + Keywords/ + Opportunities/）
- ✅ Step 2: 只填痛点库 + 关键词库（不做完整的品牌/产品库）
- ⏭️ Step 3: 跳过竞品拆解
- ✅ Step 4: 内容生态（聚焦于该话题相关的科普/教程/讨论/新闻类内容）
- ✅ Step 5: 以知识地图为核心产出（概念树 + 技术路线图 + 产业链 + 机会地图）
- ⚪ Step 6: 仅 Deep 档执行

### 特殊处理
- **概念界定**: 第一步必须明确定义这个 topic 的边界（包含什么/不包含什么/和相关概念的关系）
- 技术型 topic 需要梳理技术演进路线
- 商业型 topic 需要梳理产业链和玩家格局
- 学术/研究型 topic 需要梳理关键论文和研究者

### 字段裁剪
- 品牌/产品: 只列出与该 topic 直接相关的核心玩家及其相关产品/方案（Top 10-15 即可）
- 痛点: 聚焦于该 topic 使用者的痛点和门槛
- 关键词: 包含学术术语 + 搜索词 + 行业黑话
- 知识地图: 这是核心产出，要做最详细（可到 4-5 级深度）
- 内容: 科普文/教程/论文解读/讨论帖/新闻

### 取数策略
- 学术/技术: Google Scholar / arXiv / 论文博客 / 技术文档
- 商业应用: 行业报告 / 新闻 / 公司官网 / 产品页
- 讨论: Reddit / 知乎 / HackerNews / 专业技术论坛
- 教程: YouTube / 博客 / Documentation Site / 课程平台

## 五、模式切换速查表

| 用户输入 | 自动判定 mode | 是否需要确认？ |
|---------|--------------|-------------|
| `"美国宠物零食市场"` | industry | 否 |
| `"帮我研究一下 SaaS 行业"` | industry | 否 |
| `"拆解 OpenAI"` | brand | 否 |
| `"分析一下 Lululemon 这个品牌"` | brand | 否 |
| `"https://shop.goli.com"` | website | 否 |
| `"看看 notion.so 这个网站"` | website | 否 |
| `"GLP-1 减肥药"` | topic | 可能——如果用户实际想研究整个减肥药行业则应升格为 industry |
| `"AI Coding"` | topic | 可能——如果想研究的范围很大可升格为 industry |
| `"MCP"` | topic | 否 |

**模糊情况处理原则**: 宁可范围偏大（industry）不要偏小（topic）。用户可以在执行过程中说"缩小范围"。
