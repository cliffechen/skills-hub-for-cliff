# 全域蒸馏器 (Full-Spectrum Distiller) · 使用说明

> **Skill 名称**: `full-spectrum-distiller`
> **版本**: v2.2 | **注册名**: full-spectrum-distiller（与目录名 industry-distiller 不同，按 SKILL.md frontmatter 的 name 字段注册）
> **适用环境**: WorkBuddy 桌面应用

---

## 一、这是什么

全域蒸馏器是一个 AI 工作流 Skill。给它一个行业名、品牌名、网站 URL 或具体话题，它按固定流程自动执行全套调研，产出结构化的认知档案。

**「蒸馏」的含义**：来自化学实验室，把混合物加热蒸发再冷凝提取纯物质。放到信息领域，就是从全网散乱信息中提纯出一套完整的、有来源验证的、可迭代的知识资产。

---

## 二、它能做什么（输入 → 输出）

```
输入                              产出
─────────                        ─────────
"蒸馏 Claude"                    ─┐
"帮我研究一下 DTC 咖啡"         │   output/<slug>-OS/
"拆解 https://notion.so"       ├──→ ├─ _INDEX.md              （总导航）
"研究 GLP-1 减肥药赛道"          │   ├─ Brands/*.md            （品牌数据库 ×17字段）
                                   │   ├─ Products/*.md           （产品分类树）
                                   │   ├─ Keywords/*.md          （关键词 ×5类意图）
                                   │   ├─ Pain-Points/*          （用户痛点）
                                   │   ├─ Competitors/*          （竞品拆解 ×6-10家）
                                   │   ├─ Content/*              （内容生态 +爆款规律）
                                   │   ├─ Knowledge-Cards/*      （知识卡片）
                                   │   ├─ Opportunities/*        （机会清单 ×4类）
                                   │   └─ report.html            （可视化 HTML 报告）
                                  ─┘
```

**核心特点**：
- 单文件自包含 HTML 报告，双击即开，可分享
- 全部 .md 文件兼容 Obsidian（拖进 vault 即用）
- 每个文件末尾带复盘 prompt，可追问补漏
- 支持增量迭代：跑完后可以继续补充，不用从头来

## 四、四种模式

输入不同类型的内容，Skill 会自动裁剪执行范围：

| 你输入的 | 判定为 | 重点做什么 | 不做什么 |
|---------|--------|-----------|---------|
| 一个网址 | **website** | 深拆该网站 + 找同类站对比 | 不建完整行业品牌库 |
| 一个品牌/公司名 | **brand** | 以该品牌为中心的全套拆解 | 不过度扩展到全行业 |
| 一个行业名 | **industry** | 全量执行（最重） | 无 |
| 一个话题/概念 | **topic** | 知识地图 + 痛点 + 内容生态 | 不做竞品拆解 |

## 五、三档深度

| 触发词 | 档位 | 数据量 | 跑哪些步骤 | 耗时 | 场景 |
|-------|------|--------|-----------|------|------|
| `"快速蒸馏 XX"` / `"过一下 XX"` | Quick | Top 10-15 | Step 2 + 6 | ~5 分钟 | 快速扫摸底 |
| **不指定 / 默认** | **Standard** | **Top 20-30** | **Step 0→1→2→2.5→3→5→6 + HTML** | ~8-15 分钟 | 大多数情况 |
| `"深度蒸馏 XX"` / `"全套研究 XX"` | **Deep** | Top 50-100 | **全跑 + Step 7 情报系统** | ~20-40 分钟 | 长期跟踪 / 重度投入 |

**建议**：第一次用默认标准档，跑完觉得哪块不够深再针对性加深或补齐。

## 六、完整流程说明

### Step 0-1：初始化 + 建骨架

自动识别输入类型，匹配已有行业模板（如果之前研究过类似行业会自动复用），创建目录树和进度追踪文件 `_INDEX.md`。

### Step 2：填数据库

整个流水线最重的数据采集阶段：

| 子步骤 | 产出 | 说明 |
|--------|------|------|
| 2.1 品牌库 | `Brands/{name}.md` | 每家 17 字段（官网/创始人/价格/渠道/卖点/社媒/**用户评价引用**/**数据缺口**）|
| 2.2 产品库 | `Products/TREE.md` | 分类树 + 每品类独立文件 |
| 2.3 痛点库 | `Pain-Points/PAIN_POINTS.md` | 来自 Reddit/社区的真实抱怨，不是编的 |
| 2.4 关键词库 | 5 个 `.md` 文件 | 按 commercial/informational/comparison/review/buying-intent 分 |

**质量铁律**：宁可只列 20 个但每条都有官网/创始人/价格/社媒，也不要列 100 个只有名字的空壳。抓不到实时数据的标注 *数据截至*，绝不编造。

### Step 2.5：URL 验证（防假）

解决 AI 找网站经常出错的痛点。每个 URL 必须过三关：
1. **存活检查**：能打开？返回 200？
2. **交叉验证**：至少 2 个独立源指向同一地址
3. **内容抽样**：首页有 logo？导航正常？产品线对得上？

同时配备 Google 高级搜索语法武器库（30+ 条），用于精准定位独立站。

### Step 3：竞品拆解（12 维度）

主对象做 **12 维 deep**，其他竞品做 **8-12 维**：

| # | 维度 | 看什么 |
|---|------|--------|
| 1 | 导航结构 | = 老板的大脑，暴露利润角色分配 |
| 2 | Collection 分类 | = 已验证的高转化成交路径 |
| 3 | Tag 标签 | = 用户搜索语言 = 推荐系统逻辑 |
| 4 | SEO 结构 | 先研究头部写什么，直接模拟内容策略 |
| 5 | Blog 结构 | 行业最大流量来源 |
| 6 | Footer | 品牌想被什么关键词命中 |
| 7 | Landing Page | 转化页面模块设计 |
| 8 | 转化路径 | 怎么把访客变成客户 |
| 9 | 工具栈 | 反映成熟度和预算 |
| 10 | 社媒矩阵 | 8 平台覆盖情况 |
| 11 | 定价页 | SaaS/B2B 定价策略 |
| 12 | 移动端 | 移动用户体验 |

> 当你分析完 10 个竞品，看到的已经不是一家公司怎么运营，而是整个行业中大家都是如何赚钱的。

### Step 5：内容生态

核心理念：**不要研究一个账号，要研究 100 个账号。一次爆可能是运气，十次爆一定是规律。**

- **50+ 账号大规模采样**（YouTube/X/TikTok/小红书/Reddit/GitHub/Substack）
- **五分类法**归类（曝光型 / 涨粉型 / 收藏型 / 转化型 / 人设型）
- **Top 20 可借鉴档案**（钩子套路 / 节奏结构 / 视觉包装 / CTA 设计 / 代表作 3 条）
- **爆款规律挖掘**：沉默赢家（粉丝少但每条爆）/ 死掉的爆款 / 跨平台迁移机会

### Step 6：知识地图 + 机会

- **三级树状图**（Mermaid 格式思维导图）
- **连接关系网络图**（产业链/跨界/投资/人才流动）
- **4 类机会文件**：低竞争蓝海 / 高增速方向 / 内容空白 / Aha Moments

### Step 7：情报系统（仅 Deep 档）

- 信息源清单（Top YouTube/X/Newsletter/Reddit/Blog）
- RSS 监控清单
- 竞品周监控模板
- 自动行业周报模板

跑完这一步，数据库变成了一个能自己更新的活知识库。

### Final：HTML 报告

自动生成一份自包含 HTML 报告：
- 侧边栏锚点导航（sticky，滚动高亮当前章节）
- Chart.js 图表（散点图 / 柱状图 / 雷达图）
- 4-tab 机会切换
- 打印友好（Ctrl+P 出 PDF）
- 双击即开，可分享给任何人

## 七、跑完之后

### 查看结果

**方式一：看 HTML 报告**
双击 `output/<slug>-OS/report.html`，浏览器打开即可。含所有关键数据摘要 + 图表 + 导航。

**方式二：深入查看 .md 文件**
- 直接打开 `output/<slug>-OS/` 目录浏览
- 或把整个 `<slug>-OS/` 文件夹拖进 Obsidian vault（支持双链和图谱视图）

### 追问补漏

如果某一块不够深，两种方式：

**A. 直接说**
```
补齐 Claude 蒸馏的内容生态
深拆 ChatGPT 这个竞品（补 12 维）
加上情报系统模块
```
它会在已有成果上增量追加新文件并重新生成报告。

**B. 用内置复盘 prompt**
每个 .md 文件末尾有一段「复盘 prompt」，复制出来发给 AI 追问，它会基于已有数据继续补充。**不需要从头来**。

### 输出文件速查

```
output/<slug>-OS/
│
├── _INDEX.md                  🔑 顶层导航（从这里开始看）
├── _MAP.md                    🗺️ 知识地图（Mermaid 格式）
├── _CONNECTIONS.md            🔗 连接关系网络图
├── _REFLECTION.md             🔄 复盘 prompt 汇总
├── report.html                📊 可视化总报告（双击即开）
│
├── Brands/                    👥 品牌数据库（每家 17 字段）
├── Products/                  📦 产品分类树
├── Keywords/                  🔤 关键词（5 类意图分文件）
├── Pain-Points/               😫 用户痛点（真实抱怨）
├── Competitors/               ⚔️ 竞品拆解（12 维 deep + 横评矩阵）
├── Content/                   📱 内容生态（50+ 账号 + 爆款规律）
├── Knowledge-Cards/           🧠 知识卡片
├── Opportunities/             💡 机会清单（4 类 18+ 条）
└── Content-Sources/           📡 内容源清单（创作者 + 社区）
```

Standard 档通常产出 40-60 个 .md 文件 + 1 个 html，Deep 档可达 80-120 个文件。

## 八、适用场景

| 场景 | 怎么用 | 拿到什么 |
|------|--------|---------|
| 创业前调研 | `蒸馏 XX 行业` | 完整玩家格局 + 痛点 + 机会 + 知识地图 |
| 竞品监控 | 定期 `蒸馏 XX` + 追问更新 | 动态变化追踪（新品/功能/营销）|
| 内容创作 | 看 Opportunities + Content/patterns/ | 选题库 + 爆款公式 + 标题套路 |
| SEO / 落地页 | 看 Keywords/*.md | 用户原话语句 + 搜索意图分类，直接可用 |
| 投资研究 | `深度蒸馏 XX --deep` | 含估值/融资/ARR/护城河分析 |
| 学习教育 | 看 _MAP.md + Knowledge-Cards/ | 系统性知识框架，不是碎片信息 |
| 出海调研 | industry 模式自动覆盖中英文源 | 全球视野的竞品和内容生态 |
| 公众号写作 | 从机会清单找话题 + 从爆款规律学结构 | 有数据支撑的内容选题 |

## 九、安装方法

将本文件夹中的 `SKILL/` 目录复制到你的 WorkBuddy 用户级 skill 目录下：

```
复制目标路径:
~/.workbuddy/skills/industry-distiller/
```

确保目录下存在 `SKILL.md` 文件（这是 Skill 的入口）。WorkBuddy 会按 frontmatter 中的 `name: full-spectrum-distiller` 自动注册。

## 十、注意事项

1. **需要联网** — 流程涉及 WebSearch 搜索 + WebFetch 页面抓取
2. **输出位置** — 默认输出到当前工作空间的 `output/` 目录
3. **首次使用** — 建议先用 Quick 档试一次熟悉流程，再用 Standard 档正式跑
4. **增量更新** — 跑完后的成果可以持续追问补充，每次补充后 report.html 会自动重生成
5. **Obsidian 兼容** — 所有 `.md` 文件可直接拖入 Obsidian vault 使用
