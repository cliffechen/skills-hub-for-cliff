# 亚马逊下拉框拓词 · 美国站专用 (Amazon US Dropdown Keyword Expander)

## 这个 skill 做什么

模拟在亚马逊**美国站**搜索框输入关键词后弹出的下拉联想词，批量采集这些词，
作为亚马逊关键词调研和关键词库的来源。复刻自 `亚马逊下拉框拓词.exe`。

> 本 skill **只面向美国站**（amazon.com）。美国站已同时覆盖英语与西语用户——
> 直接输入西语种子词（如 `pañales para bebé`）即可拿到西语联想词，无需切换语言或站点。

## 原理

亚马逊搜索框的下拉联想由一个公开接口提供：
`https://completion.amazon.com/api/2017/suggestions`。
每输入一段前缀，接口返回约 10 个推荐词的 JSON。

本工具用经典的 **"字母表轰炸法 (alphabet soup)"**：
对每个种子词 `X`，依次查询 `X`、`X a`、`X b` … `X z`，把每次返回的推荐词收集、去重。
**默认开启二级拓词**：再用一级结果里词频最高的词作为新种子拓第二轮，挖更深的长尾
（如只需一级，用 `--l1-only`）。

> 说明：本工具用匿名 HTTP 请求，亚马逊对匿名请求每个前缀通常返回 3~5 条
> （真实浏览器会话能拿满 ~10 条）。默认的二级拓词往深里挖可弥补广度，总词量通常更多。

## 运行环境

- 需要 **Python 3**（3.7+）。命令视系统用 `python` 或 `python3`。
- **零第三方依赖**，仅用标准库，开箱即用。
- 脚本文件：与本 SKILL.md **同目录**下的 `expander.py`。

## 如何使用（agent 操作步骤）

1. 向用户确认**种子关键词**（可多个，英语或西语均可）。站点固定美国站，无需询问。
2. 运行与本文件同目录的 `expander.py`。把 `<DIR>` 换成本 skill 所在目录：

   ```bash
   python "<DIR>/expander.py" -k "saffron supplement" -o "<输出目录>"
   # 若 python 不可用,改用: python3 "<DIR>/expander.py" ...
   ```

   常见用法：

   ```bash
   # 多个种子词 (默认已开二级拓词)
   python expander.py -k "iphone case" "phone charger"
   # 从文件批量读取(一行一个)
   python expander.py -i seeds.txt
   # 西语种子词(覆盖美国西语市场)
   python expander.py -k "pañales para bebé" "comida para perros"
   # 只做一级拓词(更快、词较少)
   python expander.py -k "iphone case" --l1-only
   ```

3. 输出目录默认生成两个文件，**文件名带种子关键词**，一眼就能认出是哪个词的调研：
   - `时间戳_<种子词>_out.csv`：**主交付物**。词频统计表（Excel 可直接打开，utf-8-sig），
     列为 `排名,关键词,词频,发现层级,来源种子词,词数(空格分词)`，按词频降序。
     表头下方紧接两行说明，运营拿到手即可理解各列含义，无需额外解释：
     - **📖 通俗解释** — 用大白话说明每列是什么意思（如"词频"解释为"这个词在所有查询中反复出现的次数，数字越大说明越核心、越热门"）。
     - **🎯 选词建议** — 告诉运营拿到每列数据怎么用（如词频≥5 的词做核心词重点优化，层级1的词适合标题和五点，层级2的长尾词适合 PPC 精准投放，5-7 词的长尾适合精准匹配广告等）。
     **词频越高 = 该词在多个查询里反复出现，通常代表越核心/越热门**。
   - `时间戳_<种子词>_log.txt`：采集过程日志。
   - （加 `--txt` 可额外输出 `时间戳_<种子词>_out.txt` 纯关键词列表。）

   多个种子词时文件名取第一个词并标注数量，如 `..._iphone-case-etc2_out.csv`。
4. 把关键词总数与 CSV 文件路径报告给用户；CSV 里的高频词可作为重点词推荐。

## 参数说明

| 参数 | 说明 | 默认 |
|------|------|------|
| `-k, --keywords` | 种子关键词，可空格分隔多个（含空格的词用引号；英语/西语均可） | 无 |
| `-i, --input` | 种子关键词文件，一行一个（可与 -k 同用） | 无 |
| `-o, --outdir` | 输出目录 | 当前目录 |
| `--alphabet` | 后缀字母表：`az` / `az09`(含数字) / `none`(仅原词) | az |
| `--depth` | 拓词层级：`1` / `2`(用一级高频词再拓一轮) | **2** |
| `--l1-only` | 只做一级拓词（等价于 `--depth 1`） | — |
| `--l2-top` | 二级拓词时选取的一级高频种子数量(控制规模) | 20 |
| `--delay` | 每次请求间隔秒数（调大更稳、降低被限流概率） | 0.3 |
| `--limit` | 每次请求最多取词数 | 11 |
| `--txt` | 额外输出纯关键词 txt（默认只交付 csv + log） | 否 |

## 注意事项

- **仅美国站**：本 skill 已精简为美国站专用，不支持其他站点。
- **联想词随时间变化**：同一种子词不同时间跑结果会有差异（新品/季节性热词进出），属正常现象。
- **反爬限流**：默认开启二级拓词，请求量较大（一级约 27 次 + 二级约 l2-top×27 次）；
  种子词较多时建议增大 `--delay`（如 0.5）；若连续请求失败，放慢速度或稍后重试，或临时用 `--l1-only`。
- **合规**：本工具为抓取行为，使用前请自行评估是否符合亚马逊使用条款及自身使用场景。
