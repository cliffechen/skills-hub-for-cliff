# Message Hierarchy Reference

Use this reference whenever the user asks for Amazon supplement main image logic, secondary image copy, A+ page content, or layout-driven copy. This file captures the ecommerce image principle: information can be rich if the viewer knows what to read first.

## Core Principle

Do not stack selling points as equals. Build a decision path:

1. Product identity: tell the shopper what this is.
2. User benefit: tell the shopper why it matters to them.
3. Proof and facts: use ingredient, dose, format, testing, or certification to make the benefit credible.
4. Purchase uncertainty: answer the objection or concern that could stop the order.

For Amazon US supplements, do not use refund promises, guaranteed outcomes, disease claims, before/after claims, or exaggerated result language as the "uncertainty reducer." Use safer trust signals such as clear Supplement Facts, third-party testing, dietary attributes, serving count, use directions, allergen statements, packaging accuracy, and brand transparency.

## Product Language to User Language

Product language is what the formula contains. User language is what the shopper can understand in one glance.

Use this transformation:

```text
Product fact -> Shopper meaning -> Safer ecommerce line -> Proof anchor
```

Examples:

| Product language | User-language direction | Safer Amazon supplement wording |
|---|---|---|
| Magnesium glycinate 400 mg | A recognizable daily magnesium format | "Daily magnesium support in a gentle glycinate form" |
| Omega-3 DHA + EPA | A familiar nutrient support story | "Omega-3 support without adding another softgel" |
| 90 capsules / 45 servings | Less repurchase uncertainty | "45 servings per bottle" |
| No proprietary blend | Easier to trust the label | "Every active amount is listed" |
| Third-party tested | Reduces quality concern | "Tested for identity and purity" if proof is supplied |
| Vegan capsules | Fit with a diet preference | "Vegan capsule format" |

Do not over-translate into a medical result. "Supports normal immune function" is safer than "keeps you from getting sick." "Supports calm focus" may still need substantiation and should avoid anxiety language.

## One Core Memory Point

Each main image sequence or A+ page needs one phrase the shopper can remember. Other facts should serve that phrase.

Good memory point structure:

```text
[Product identity] + [plain user benefit or reduced hassle]
```

Examples:

- "Magnesium glycinate for daily relaxation support"
- "Omega-3 support without a separate fish oil softgel"
- "A transparent probiotic blend for daily digestive support"

Avoid making the hero headline carry every proof point. Move dose, certifications, dietary claims, and mechanism terms to secondary layers.

## Simple vs Scientific Language

Use simple language to win attention and scientific language to earn trust.

Recommended ratio:

- Search/gallery hero layer: 70% simple user language, 30% proof language.
- Secondary image or A+ module headline: 60% simple user language, 40% ingredient/proof language.
- Formula facts, comparison, or mechanism module: 40% simple user language, 60% precise science/facts.

Layering pattern:

1. Headline: plain buyer benefit.
2. Subhead: product identity plus one proof anchor.
3. Callouts: exact dose, form, serving count, certification, or dietary attribute.
4. Footnote/body: compliance boundaries, source, or clarifying detail.

## Visual Hierarchy Rules

- Let the largest text answer "what is it?"
- Let the second layer answer "why buy?"
- Let product packaging prove "what will I receive?"
- Let small callouts support, not compete with, the hero line.
- Use numbers only when they strengthen credibility. Do not turn every number into a headline.
- If every element wants to be first, remove or demote one.

## Amazon Adaptation

This ecommerce logic applies differently by placement:

- First Amazon main image: use it for product identity through real packaging only. Do not add benefit copy, badges, or overlays.
- Secondary gallery images: use the full hierarchy with visible copy, proof, and objection handling.
- A+ pages: build a scrollable story. Start with identity and benefit, then educate, prove, compare, and answer objections.
