# HTML 精简报告规范

## 必选区块

1. Header
2. KPI 卡片
3. 多维分布表
4. 消费者痛点卡片
5. 站外网络信号与成分反馈（启用 xCrawl 时必选）
6. 供需缺口/机会空白
7. 品牌/竞品格局
8. 进入壁垒评估
9. Go / No-Go评分卡
10. 策略建议
11. Footer

## 站外网络信号区块

启用 xCrawl 时，HTML 精简报告必须包含：

- 搜索词数、抓取页面数、覆盖平台
- 平台情绪分布：正面/中性/负面
- 站外需求信号
- 成分真实反馈
- 风险/安全信号，统一标注“需另行评估”

该区块只做市场语境补充，不输出医学或合规结论。

## 样式约束

- 使用 CSS 变量统一颜色
- 容器 `max-width: 960px`
- KPI 用网格布局
- 表格可读、可响应式缩放
- Footer 必须写清数据来源、采集日期、MD 与 XLSX 文件名

## 参考实现

见 `examples/bluetooth-speaker/` 中的 HTML 精简报告文件。

## 常见问题

- 不要只给"一段话 + 一个表格"的简版
- 不要出现"约""少量""大概"这类模糊词替代数字
- HTML 与 Markdown / Excel 必须同源，不允许三份文件数字对不上
