# 成分拆解与合规审查

## 概述

将品牌、产品、ASIN 或成分拆解为结构化的成分档案，然后围绕成分堆栈评估知识产权和合规风险。本技能适用于以保健品、健康产品和成分驱动型电商调查为主的可重复研究工作流。

当用户需要了解以下内容时使用本技能：
- 某品牌或产品可能含有哪些成分
- 哪些成分看起来是品牌化的、已注册商标的、专有的或受供应商控制的
- 标签上的某个术语作为通用描述使用是否比直接引用品牌成分更安全
- 某个配方或宣称组合是否会产生 FDA 或亚马逊合规风险
- 在上架、换标或仿制前需要准备哪些文件或检查

正文保持简洁。按需加载参考文件。

## 工作流程

每次新调查开始时，阅读 `references/prompt-library.md`，选取最接近的起始模板作为研究框架。

1. 标准化输入信息。
2. 构建成分拆解。
3. 将每种成分分类为：通用成分、品牌成分、身份不明或高风险。
4. 执行商标、专利和商业化检查。
5. 当产品为口服类、健康定位或以膳食补充剂销售时，执行 FDA 和亚马逊政策检查。
6. 跨平台内和平台外来源交叉验证发现。
7. 输出结构化报告，包含已确认事实、强信号、待确认问题和下一步需获取的文件。

按需阅读以下参考文件：
- `references/prompt-library.md` — 高命中率起始提示词和请求框架。
- `references/source-strategy.md` — 信息源优先级和 MCP 选择策略。
- `references/ingredient-breakdown.md` — 成分拆解工作流。
- `references/ip-and-proprietary-risk.md` — 商标和专利关联成分分析。
- `references/compliance-and-marketplace-risk.md` — FDA 和亚马逊风险审查。
- `references/report-template.md` — 最终报告输出格式。

## 标准化输入信息

支持以下起始输入类型：
- `ASIN`
- `品牌名称`
- `产品标题`
- `成分名称`
- `Supplement Facts（营养成分表）` 或成分列表文本
- `Listing 截图`

将输入转化为目标档案，包含：
- 标准化的产品或品牌名称
- 销售平台和区域
- 可能的产品类别
- 可见的成分和宣称
- 每个非显而易见成分的同义词列表

在判定风险之前，务必先构建成分别名列表，包括：
- 拼写变体
- 缩写
- 拉丁名或植物学名称
- 盐类、酯类、水合物、提取物和发酵形式
- 带有 `TM`、`R`、`(R)` 或特殊大写格式的品牌形式
- 可能位于品牌成分下方的通用描述

## 信息源策略

参考 `references/source-strategy.md`。

默认信息源优先级：
1. 直接产品和市场数据
2. 品牌所有者、制造商或供应商官方页面
3. 美国官方监管或知识产权来源
4. 专利和文献数据库
5. 发现类来源，如 AnySearch（主要）和 Exa（备用）

有目的地使用 MCP 工具：
- 使用 `mcp__sorftime_mcp__product_detail`、`product_search`、`product_reviews`、`product_report`、`product_traffic_terms` 或相关 Sorftime 工具来丰富亚马逊端的产品、类目、评论和关键词背景。
- 使用 `mcp__AnySearch__search` 和 `mcp__AnySearch__batch_search` 作为主要的站外发现工具 —— 先使用 `get_sub_domains` 进行特定领域垂直搜索（健康、法律、商业、学术、知识产权），当摘要信息不足时使用 `extract` 获取完整页面内容。
- 仅当 AnySearch 无法找到所需的所有者页面、产品说明页、已归档宣称或小众成分参考时，才使用 `mcp__exa_mcp__web_search_exa` 和 `mcp__exa_mcp__crawling_exa` 作为备用。

Sorftime 用于市场背景，AnySearch 用于主要站外发现，Exa 用于备用发现。任何工具都不能替代法律或监管结论所需的官方验证。

## 成分拆解规则

参考 `references/ingredient-breakdown.md`。

对每种成分或疑似成分，将其归入以下四类之一：
- `通用成分（Generic ingredient）`
- `品牌成分（Branded ingredient）`
- `疑似专有成分（Likely proprietary ingredient）`
- `身份不明（Unclear identity）`

同时记录：
- 成分功能/角色
- 可见剂量或相对含量排序
- 来源生物体或原材料
- 工艺线索
- 活性标志物或标准化线索
- 与该成分关联的宣称用语

如果产品信息不完整或标签模糊不清，不要默默猜测。标记缺失字段并在明确标注不确定性的情况下继续调查。

## 知识产权与专有成分分析

参考 `references/ip-and-proprietary-risk.md`。

对每个可疑成分检查：
- 商标使用或来源标识使用
- 品牌成分采购风险
- 专利或自由实施（freedom-to-operate）信号
- 独家供应或专有工艺描述
- 如果用户试图推出类似产品的仿制风险

高风险信号包括：
- 成分名称带有 `TM`、`R` 或 `(R)` 标记
- 所有者将该成分宣传为独家、专利、临床研究支持或专有
- 同一成分名称出现在多个转售商处，但只有一个明显的来源所有者
- 宣称组合依赖于独特的工艺、提取方式、标准化方法或成分组成故事

不要指导用户如何规避专利或未经授权使用他人的商标。保持在风险识别、差异化方案建议和文件请求层面的分析。

## FDA 和亚马逊审查

当产品为膳食补充剂、健康产品或宣称敏感的口服类产品时，参考 `references/compliance-and-marketplace-risk.md`。

至少检查：
- 成分状态或新膳食成分（NDI）信号
- 疾病宣称和药物宣称用语
- 与可见成分表的一致性
- 亚马逊受限产品风险暴露
- 是否需要 TIC、COA、cGMP、标签图片或供应商文件

区分以下问题：
1. 成分本身是否存在明显高风险？
2. 围绕成分的用语是否存在高风险？
3. 即使成分本身未被禁用，产品是否可能触发亚马逊的文件要求或政策审查？

## 输出要求

参考 `references/report-template.md`。

始终包含：
- 简短的执行摘要
- 标准化的输入信息和实际审查范围
- 成分拆解表格
- 商标、专利、合规和市场审查的风险评级
- 每个主要结论的置信度标签
- 能最快降低不确定性的 3-5 个下一步文件或检查建议

## 安全守则

不要将本技能呈现为法律建议。
- 不要断言某个产品确定不侵权。
- 不要指导用户复制专利工艺或品牌成分体系。
- 不要假设缺少商标符号就意味着该术语是通用的。
- 不要仅凭单一市场页面就得出决定性结论。
- 除非有官方来源明确支持，否则不要声称某膳食补充剂已获得 FDA 批准。

如果用户使用中文提问，以中文回复，但在有用的地方保留成分名称、商标、法规名称和 URL 的英文原文。
