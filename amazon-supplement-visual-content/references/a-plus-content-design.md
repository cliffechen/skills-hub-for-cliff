# A+ Content Design Reference

Use this reference when the user asks for A+ page content, A+ module planning, A+ copy, brand story, comparison charts, FAQ, or content architecture. This is for content design, not image generation, unless the user explicitly asks for visuals.

Official starting points:
- Amazon Seller Central A+ content guidelines: https://sellercentral.amazon.com/help/hub/reference/external/GGW8U76SSNTRTBX7
- Amazon A+ Content overview: https://sell.amazon.com/tools/a-content
- Amazon A+ Content guide: https://sellercentral.amazon.com/help/hub/reference/external/GLG4RQK2Y2RJADU4

## What A+ Should Do

A+ content should help shoppers make a more informed purchase decision by adding richer product and brand information than the bullet points can hold. Amazon describes A+ as a way to showcase products and brand story with enhanced images, customized text placements, comparison charts, videos, hotspots, Q&A modules, and related rich formats.

For supplements, treat A+ as a trust-building education path:

1. Clarify product identity.
2. Translate formula facts into user-understandable benefits.
3. Explain mechanism conservatively.
4. Show exact label facts and proof.
5. Answer practical questions.
6. Reinforce brand trust and product fit.

## Recommended Supplement A+ Sequence

Use 5 to 7 modules for a standard A+ page unless the user requests otherwise.

| Order | Module role | Job | Best content |
|---:|---|---|---|
| 1 | Hero positioning | Establish the core memory point | Product identity + user-language benefit + one proof anchor |
| 2 | Ingredient translation | Turn formula into shopper meaning | "What it is" and "why it matters" pairs |
| 3 | Mechanism education | Explain support area without disease claims | Normal body-function support, simple diagram notes |
| 4 | Formula facts | Reduce skepticism | exact Supplement Facts, dose, form, serving count |
| 5 | Routine fit | Show how to use it | serving directions, timing, lifestyle fit, warnings |
| 6 | Proof/trust | Support quality claims | testing, certifications, sourcing, manufacturing proof |
| 7 | FAQ/comparison | Remove objections | own-brand comparison, who it is for, fit boundaries |
| 8 | Brand story | Add memory and trust | transparent formulation philosophy, label clarity, values |

Do not use every module if the evidence is thin. It is better to have fewer stronger modules than repeated generic wellness copy.

## Module Copy Rules

- Write customer-facing A+ copy in English for Amazon US.
- Keep headlines plain and memorable; use scientific terms in subheads or body.
- Put only one evidence anchor in each module.
- Keep body copy to 1 to 3 short sentences.
- Use exact Supplement Facts terms for dose, form, units, and serving size.
- Keep claims consistent with title, bullets, description, images, alt text, and label.
- Add a compliance note for every support claim.

## Product-Language Translation for A+

For each module, include a translation decision:

```text
Product fact: [ingredient/dose/form/proof]
User meaning: [plain benefit or reduced hassle]
A+ expression: [headline/subhead/body]
Proof anchor: [label, COA, certification, study, or user-supplied evidence]
```

Example:

```text
Product fact: 400 mg magnesium glycinate per serving
User meaning: daily magnesium support in a recognizable, gentle form
A+ expression: Magnesium support, clearly labeled
Proof anchor: Supplement Facts panel
```

## A+ Compliance Guardrails

Avoid:

- disease treatment, prevention, cure, symptom relief, or drug-like claims
- guaranteed results, before/after promises, time-bound outcomes
- price, discount, coupon, shipping, refund, or warranty language
- review quotes, star ratings, rankings, "best seller", or "No. 1" claims unless Amazon rules and evidence clearly allow the exact usage
- QR codes, external URLs, external contact information, or redirect language
- competitor attacks or unverified superiority claims
- badges or certifications the user has not supplied
- repeating medical or disease-adjacent keywords in alt text

Comparison charts should prefer own-brand product comparisons or neutral product-fit comparisons. If the user requests competitor comparison, require strong evidence and keep language factual, non-defamatory, and policy-reviewed.

## Review Pass

Before final output:

1. Check whether the opening module has one clear memory point.
2. Check whether every scientific term has a plain-language reason.
3. Remove repeated claims across modules.
4. Remove unsupported proof, medical implications, or purchase promises.
5. Mark facts that require user evidence before publication.
